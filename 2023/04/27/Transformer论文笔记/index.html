Transformer论文笔记

出处：NIPS'17

原文：Attention Is All You Need

代码：tensor2tensor

摘要

在主流的序列转录模型中，一般是依赖于比较复杂的包含encoder-decoder架构的循环或卷积神经网络。在最好的模型中，在编码器和解码器之间会使用注意力机制。本文提出了一个简单的架构Transformer，该架构仅依靠注意力机制而未使用循环或卷积神经网络。本论文的实验是基于机器翻译的任务，取得了较好的效果。

导言

当前最常用的序列转录模型有RNN、LSTM、GRU，在其中有两个比较主流的模型，分别是语言模型recurrent language model和编码器-解码器架构encoder-decoder。在RNN中，序列的计算是从左往右一步一步做，以此达到时间序列的效果。每一步计算得到一个输出，也叫做隐藏状态，每个由前面的状态和当前本身状态决定的，即通过历史信息和当前信息的计算得到当前节点输出。但是它也存在一些问题：由于时序计算有明确的顺序，所以计算过程难以并行化；由于历史信息是一步一步传递的，所以一旦时许较长，那么较早的历史信息会被遗忘，如果全部存储则内存开销很大。

而注意力机制在之前已经被用在编码器-解码器架构中了，其重要作用是解决如何把编码器的东西有效地传给解码器。但是注意力机制还是得和RNN一起使用。

而本文提出的Transformer将不再使用循环神经层，而是纯使用注意力机制，从而可以实现很好的并行。

相关工作

之前的研究会使用卷积神经网络来替代循环神经网络，从而减少时序的计算。但如果需要对较长的序列建模，那么需要很多次卷积层才能使得序列相隔较远的两个节点建立联系。但如果使用注意力机制，每一次都可以看到所有的节点，相对来说就不会存在这个问题。卷积还有一个比较好的地方，在于它可以形成多个输出通道，每一个输出通道可以识别不一样的模式。而本文提出的多头注意力机制Muti-Head Attention可以模拟RNN这个效果。

本文使用到的一个重要的注意力机制是自注意力机制Self-attention，是一种将单个序列的不同位置联系起来以计算该序列的表示的注意机制。

模型架构

大多数较好的神经序列转导模型都具有编码器-解码器结构。假设有一段输入表述序列，编码器会将这个序列处理成向量表示输出。而解码器会根据编码器的输出生成一个长度为m的序列，且序列中每个节点是一个一个生成的，因为在每个步骤中，模型都是自回归的，在生成下一个步骤时，使用先前生成的符号作为附加输入。

Transformer遵循这个整体架构，使用堆叠的自注意力和point-wise、编码器和解码器的完全连接层，具体如下图：

Encoder and Decoder Stacks

编码器：编码器由六个相同的层堆叠而成，每一层有两个子层。第一个子层是一个多头注意力机制，第二个子层是一个简单的MLP。我们在每两个子层周围使用残差连接residual connection，然后进行层归一化。每一层的工作可以用公式表示为，由于这里需要使用残差连接，所以这里简单地把MLP中所有层的维度都设为512维。所以总的来说整个encoder只有两个参是可调的，分别是层数N和MLP的维度。

关于残差连接和Layer Norm的解释在李宏毅老师的PPT中有着非常直观的体现：

解码器：解码器也是由六个相同的子层构成的。除了编码器中使用过的两个子层，解码器中还引入了第三个子层，它对编码器层的输出执行多头注意力。与编码器类似，在每个子层周围使用残差连接，然后进行层规范化。我们知道自注意力机制是可以一下子看见整个输入的，但是解码器的输入是自回归的，不能知道当前节点后来的节点是什么，所以我们需要将当前节点后的节点进行掩膜处理Mask，从而使得输入都是已知计算出的结果（训练和预测的时候行为一致）。具体的Mask操作是将需要掩膜的值乘上一个非常大的负数，从而使得这个值在后续算softmax的时候的值几乎为0，从而对结果不产生影响。

Attention

原文对Attention的描述如下：

An attention function can be described as mapping a query and a set of key-value pairs to an output, where the query, keys, values, and output are all vectors. The output is computed as a weighted sum of the values, where the weight assigned to each value is computed by a compatibility function of the query with the corresponding key.

可以结合李宏毅老师的自注意力PPT理解：

注意函数可以描述为将查询和一组键值对映射到输出，其中查询、键、值和输出都是向量。输出是作为值的加权和计算的，其中分配给每个值的权重是由查询与相应键的兼容性函数（相似度）计算的。之所以叫自注意力，是因为每个的q、k、v都是它本身。

Scaled Dot-Product Attention

注意力的输入是维度均为的query和key，以及维度是的value（所以输出维度也是）。将query和每个key做内积dot product，然后除上，通过一个softmax得到value的权重。

在实际中，使用矩阵进行运算，所有query放入矩阵Q，key和value放入矩阵K和V，则矩阵计算的过程如下： 一般有两种比较常见的注意力机制，分别是是加型的注意力additive attention和内积注意力dot-product(multiplicative) attention。前者处理query和key不等长的情况，后者和本文中注意力类似，不同之处在于本文的attention除以了。

当的值不是很大时，这一步运算看起来并没有很大影响。但是当较大，即query和key的维度较高时，二者点积的结果可能比较大，从而导致每次运算相对的差距会比较大。如果有一个值算出来结果很大，那么它的softmax值会接近于1，而剩下的值更加接近0。这个时候如果要算梯度，会发现梯度是很小的。因为softmax的作用其实就是希望置信的地方可以接近于1，不置信的地方可以接近于0。所以梯度较小的话，程序会跑不动（原文：pushing the softmax function into regions where it has extremely small gradients）。而之前说过维度是512，是较大的，所以这么做是合理的。

Applications of Attention in our Model

多头注意力在Transformer中有三个使用：

  • 在“Encoder-Decoder Attention”层中，query来自前一个decoder，而记忆的key和value来自encoder的输出。这允许decoder中的每个位置都参与输入序列中的所有位置，这也是模仿了sequence-to-sequence模型中典型的机制。
  • encoder包含自注意力层。在自注意力层中，所有的query、key和value都来自同一个地方，在这种情况下，是encoder中前一层的输出。encoder中的每个位置都可以处理编码器前一层中的所有位置。
  • 类似地，decoder中的自注意层允许encoder中的每个位置注意到decoder中的所有位置直至并包括该位置。我们需要防止decoder中的向左信息流以保持自回归特性。我们通过屏蔽(设置为−∞)softmax输入中对应于非法连接的所有值来实现缩放点积注意力。

Position-wise Feed-Forward Networks

encoder和decoder的每一层都包涵一个全连接前向网络，每一个节点都会分开做相同的运算。这个神经网络包含两个线性变换和一个ReLU激活函数，内层的网络层会把512维的输入扩充到2048维：

所以可以在整体架构上对Transformer和RNN进行一下比较，看看二者分别是如何传递历史信息的。在Transformer中，经过自注意力层后，每个MLP的输入或者说自注意力层的输出其实已经因为自注意力的特点而含有了该节点之前的历史信息；而在RNN中，每个MLP的输出会作为下一个MLP运算的输入，当然MLP的参数是相同的，从而也做到历史信息的传递。

Embeddings and Softmax

与其他序列转导模型类似，我们使用学习embedding将输入token和输出token转换为维度的向量。我们还使用通常学习的线性变换和softmax函数将解码器输出转换为预测的下一个token的概率。在我们的模型中，我们在两个嵌入层和pre-softmax线性变换之间共享相同的权重矩阵。在embedding层，我们将这些权重乘上。

乘这个量的原因，没听懂李沐在说什么，借鉴弹幕的总结：

  1. L2 Norm会将向量的所有值归一化
  2. 维度越大的向量归一化后，其单个值就越小
  3. 而时序信息是递增的整数（后面的Positional Encoding）
  4. 为了让它们的规模相匹配，故而乘了一个根号d给前面

下面是Chat老师的解释：

在计算嵌入向量权重时，Transformer 使用了一个特殊的技巧，即将权重乘上根号 d。这个技巧被称为 "sqrt d" 技巧，其中 d 是嵌入维度。这个操作的目的是使得嵌入向量的权重更加稀疏，从而减少计算量和存储量。具体来说，乘上根号 d 的权重通常只有大约 1% 的值是接近 1，而其他值都非常小，这可以大大减少计算量和存储需求。需要注意的是，sqrt d 技巧只在 embedding 层中使用，而在后面的层中仍然使用传统的权重表示。

Positional Encoding

由于我们的模型不包含递归和卷积，为了使模型利用序列的顺序，我们必须注入一些关于序列中标记的相对或绝对位置的信息。为此，我们在编码器和解码器堆栈底部的输入嵌入中添加了“位置编码”。位置编码与嵌入具有相同的维数d，因此两者可以相加。位置编码有很多选择，有习得的和固定的。这里用的是不同频率的sine和cosine函数：
